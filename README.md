# DocuRAG — OpenStax Psychology 2e Q&A

RAG system that answers questions from the OpenStax Psychology 2e textbook with citation-verifiable responses. Built for the NVIDIA hackathon.

- Embedding: sentence-transformers (three tiers — fast / balanced / deep)
- Vector store: PostgreSQL + pgvector
- Generation: NVIDIA API (`meta/llama-3.1-8b-instruct`)
- Output: `submission.csv` with `ID`, `context`, `answer`, `references`

---

## Prerequisites

- Python 3.10+
- PostgreSQL 14+ with the `pgvector` extension
- NVIDIA API key — get one free at [build.nvidia.com](https://build.nvidia.com)

## Setup

### 1. Clone and install

```bash
git clone https://github.com/Goldenmist00/DocuRag.git
cd DocuRag

python -m venv venv
# Windows
venv\Scripts\activate
# Linux / Mac
source venv/bin/activate

pip install -r requirements.txt
```

### 2. Environment variables

```bash
cp .env.example .env
```

Edit `.env` and fill in:

```
NVIDIA_API_KEY=nvapi-...
POSTGRES_PASSWORD=your_postgres_password
```

### 3. PostgreSQL setup

Make sure PostgreSQL is running and the `pgvector` extension is installed, then:

```bash
python scripts/setup_postgres.py
```

This creates the `rag_db` database, enables `pgvector`, and creates the `document_chunks` table.

### 4. Add the PDF

Download the OpenStax Psychology 2e PDF:

```
https://assets.openstax.org/oscms-prodcms/media/documents/Psychology2e_WEB.pdf
```

Save it as:

```
data/raw/openstax_psychology_2e.pdf
```

## Running the pipeline

### Step 1 — Chunk the PDF

```bash
python scripts/test_pdf_chunking.py data/raw/openstax_psychology_2e.pdf
```

Chunks are saved to `data/processed/chunks.jsonl`.

### Step 2 — Embed and store

```bash
python scripts/embed_and_store.py --tier balanced --clear --create-index
```

This embeds all chunks (uses cache on re-runs) and stores them in PostgreSQL.

Tier options:

| Tier | Model | Dimensions | Speed |
|------|-------|-----------|-------|
| `fast` | all-MiniLM-L6-v2 | 384 | ~13 chunks/s |
| `balanced` | BAAI/bge-base-en-v1.5 | 768 | ~3 chunks/s ⭐ |
| `deep` | BAAI/bge-large-en-v1.5 | 1024 | ~1 chunks/s |

### Step 3 — Generate submission

```bash
python scripts/generate_submission.py
```

Reads `data/queries.json`, runs the full RAG pipeline for every query, writes `submission.csv`.

Options:

```bash
# Dry run — retrieval only, no API calls
python scripts/generate_submission.py --dry-run

# Resume a partial run
python scripts/generate_submission.py --resume

# Override top-k
python scripts/generate_submission.py --top-k 7
```

## Output format

`submission.csv` columns:

| Column | Description |
|--------|-------------|
| `ID` | `query_id` from `queries.json` |
| `context` | Retrieved passage texts (newline-separated) |
| `answer` | LLM-generated answer with inline citations |
| `references` | JSON string: `{"sections": [...], "pages": [...]}` |

## Live demo API (optional)


Endpoints:

- `POST /query` — answer a question
- `GET  /health` — liveness check
- `GET  /stats` — vector store statistics
- `GET  /docs` — Swagger UI

Example:

```bash
curl -X POST http://localhost:8000/query \
  -H "Content-Type: application/json" \
  -d '{"question": "What is classical conditioning?", "top_k": 5}'
```

## Troubleshooting

**PostgreSQL connection refused**
Make sure PostgreSQL is running: `pg_ctl start` or check your service manager.

**pgvector not found**
Install it: `CREATE EXTENSION vector;` in psql, or follow [pgvector install guide](https://github.com/pgvector/pgvector).

**NVIDIA API 401**
Check your key in `.env`. Keys start with `nvapi-`.

**Out of memory during embedding**
Use `--tier fast` or reduce batch size in `config.yaml`.

**Re-run embedding from scratch**
```bash
python scripts/embed_and_store.py --tier balanced --clear --create-index
```
